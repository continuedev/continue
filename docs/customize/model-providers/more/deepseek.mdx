## Configuration

### Basic Configuration

<Tabs>
  <Tab title="YAML">
  ```yaml title="config.yaml"
  name: My Config
  version: 0.0.1
  schema: v1

  models:
    - name: DeepSeek Chat
      provider: deepseek
      model: deepseek-chat
      apiKey: <YOUR_DEEPSEEK_API_KEY>
    - name: DeepSeek Reasoner
      provider: deepseek
      model: deepseek-reasoner
      apiKey: <YOUR_DEEPSEEK_API_KEY>
  ```
  </Tab>

## DeepSeek Model Capabilities

DeepSeek models support various capabilities that you should be aware of when configuring:

<Card title="Supported Models & Features">

  | Model             | Tool Use | Image Input | Context Window | Max Output | Reasoning |
  | :---------------- | :------- | :---------- | :------------- | :--------- | :-------- |
  | DeepSeek Chat     | Yes      | No          | 128k           | 8k         | No        |
  | DeepSeek Reasoner | Yes      | No          | 128k           | 64k        | Yes       |

</Card>

<Info>

  **FIM (Fill-in-Middle) Model**: DeepSeek supports FIM completion through the beta endpoint `https://api.deepseek.com/beta/completions`. Its high latency may be impractical for smooth autocompletion tasks.

  **Reasoning Model Limitations**: DeepSeek Reasoner models ignore `temperature`, `top_p`, `frequency_penalty`, and `presence_penalty` parameters. They use deterministic reasoning logic instead.

</Info>

## Advanced Configurations

<Tabs>
  <Tab title="DeepSeek Chat">
  ```yaml title="config.yaml"
  name: My Config
  version: 0.0.1
  schema: v1

  models:
    - name: DeepSeek Chat
      provider: deepseek
      model: deepseek-chat
  ```

### Proxy and Custom Deployment Configuration

Continue intelligently selects between DeepSeek's beta and standard API endpoints based on your API base URL:

- **Official DeepSeek API** (`api.deepseek.com`): Uses beta endpoints (`beta/chat/completions`, `beta/completions`)
- **Custom deployments & proxy providers**: Uses standard endpoints (`chat/completions`, `completions` with suffix support)

#### Manual Endpoint Selection

If your proxy forwards requests to the official DeepSeek API but uses a different hostname, you can manually control endpoint selection:

<Tabs>
  <Tab title="YAML">
  ```yaml title="config.yaml"
  models:
    - name: DeepSeek via Proxy
      provider: deepseek
      model: deepseek-chat
      apiKey: <YOUR_DEEPSEEK_API_KEY>
      apiBase: https://your-proxy.com/deepseek/  # Your proxy URL
      useBetaEndpoints: true  # Manually enable beta endpoints
  ```
  </Tab>
  <Tab title="JSON (Deprecated)">
  ```json title="config.json"
  {
    "models": [
      {
        "title": "DeepSeek via Proxy",
        "provider": "deepseek",
        "model": "deepseek-chat",
        "apiKey": "<YOUR_DEEPSEEK_API_KEY>",
        "apiBase": "https://your-proxy.com/deepseek/",
        "useBetaEndpoints": true
      }
    ]
  }
  ```
  </Tab>
</Tabs>

<Info>

  **When to use `useBetaEndpoints: true`**:
  - Your proxy is a simple forwarder to `api.deepseek.com` (e.g., Nginx reverse proxy)
  - You want access to DeepSeek's beta features (prefix completion, FIM beta endpoint)
  
  **When to use `useBetaEndpoints: false`**:
  - Your proxy uses standard OpenAI-compatible endpoints (most API aggregators)
  - You're using local deployments (Ollama, vLLM, LM Studio)
  - You experience 404 errors with beta endpoints

</Info>
