---
title: "Headless Mode"
---

Run `yt -p "your prompt"` to execute a single task without an interactive session. The agent runs to completion and prints its response to stdout.

```bash
yt -p "Generate a conventional commit message for the staged changes"
```

## Piping

Pipe data into `yt` to include it as context:

```bash
git diff --staged | yt -p "Write a commit message for this diff"
cat error.log | yt -p "Explain what went wrong"
curl -s https://api.example.com/health | yt -p "Is this healthy?"
```

Pipe the output into other tools:

```bash
yt -p "Generate a commit message" | git commit -F -
yt -p "List all TODO comments in src/" --silent > todos.txt
```

## Output options

| Flag | Effect |
|------|--------|
| `--silent` | Strip `<think>` tags and excess whitespace from output |
| `--format json` | Output structured JSON |

`--silent` is useful when piping output into another program. `--format json` is useful for parsing results programmatically.

## Tool permissions

In headless mode, tools that would normally prompt for approval (`ask` permission) are automatically excluded — there's no one to approve them. To enable write operations, explicitly allow them:

```bash
# Allow the agent to write files
yt -p "Fix the type errors in src/" --allow Write --allow Edit

# Allow everything
yt -p "Set up the project" --allow "*"
```

See [tool permissions](/cli/tool-permissions) for the full policy system.

## Authentication in CI

Set `CONTINUE_API_KEY` instead of running `yt login`:

```bash
export CONTINUE_API_KEY=your-key-here
yt -p "Review the PR for security issues"
```

Get an API key from [Yuto Agentic → Settings → API Keys](https://yutoagentic.dev/settings/api-keys). Store it as a secret in your CI provider.

## Resume in headless mode

Resume a previous session without interactive input:

```bash
yt -p --resume
```

This replays the previous session's history, which can be useful for inspecting past results in automation.

## Examples

```bash
# Git hooks
yt -p "Review staged changes for obvious bugs" --silent

# CI pipeline — allow file writes for auto-fix
yt -p "Fix all ESLint errors in src/" --allow Write --allow Edit --allow Bash

# Generate docs from code
yt -p "@src/api/ Generate OpenAPI documentation for these endpoints" --silent > api-docs.yaml

# Use a specific assistant
yt -p "Triage this error log" --config my-org/error-triage-assistant
```
