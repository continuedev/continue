The Apply feature uses a specific context to perform its function:

1. **Original Code**: The code that currently exists in your file
2. **Generated Code**: The new code that was generated by Chat

The Apply model receives both of these pieces of context and creates a precise diff that can be applied to your file. No additional context is provided to the Apply model.

## Context Window Considerations

One of the key benefits of using a dedicated Apply model is that it can apply diffs faster than a frontier model. The Apply model is specifically designed to focus on the changes needed, rather than understanding the entire codebase.

This makes Apply particularly useful for:

- Developers wanting to move fast
- Large files
- Complex code structures
- Files with many dependencies

By using a specialized Apply model like Morph, you can ensure that your code changes are integrated seamlessly, even when working with large or complex files.
