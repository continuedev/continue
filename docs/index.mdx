---
title: "What is Continue?"
---

Continue runs AI checks on every pull request. Each check is a markdown file in your repo that shows up as a GitHub status check — green if the code looks good, red with a suggested fix if not.

<video autoPlay muted loop playsInline className="rounded-xl" src="https://continue.dev/videos/demo.mp4" />

## Start Here

New to Continue? Follow these three steps to get up and running.

<Steps>
  <Step title="Install Continue">
    Install the Continue extension for [VS Code](/ide-extensions/install) or [JetBrains](/ide-extensions/install).
  </Step>
  <Step title="Write your first check">
    Create a `.md` file in `.continue/checks/` to define what to check. See [Write Your First Check](/checks/quickstart) for a step-by-step guide.
  </Step>
  <Step title="Run checks on a PR">
    Open a pull request to see your checks run automatically. Continue reports each check as a GitHub status — green for pass, red for fail with a suggested fix.
  </Step>
</Steps>

## How It Works

<Info>
**Think of checks as automated code reviewers.** Just like a human reviewer might flag missing tests or security issues, a check does this automatically on every PR.
</Info>

A check has three parts:

| Part | What it is | Example |
|------|-----------|---------|
| **Frontmatter** | Metadata (`name` + `description`) | `name: Security Review` |
| **Prompt** | Instructions for the AI | "Flag hardcoded secrets..." |
| **Pass/Fail logic** | When to fail, what to suggest | Fail if API key found |

**The flow:**

```
Pull Request Opened
       │
       ▼
  Continue reads
  your .checks/ files
       │
       ▼
  AI reviews the diff
  against each check
       │
       ├── PASS → Green ✅ on GitHub
       │
       └── FAIL → Red ❌ on GitHub
                  + Suggested fix
```

<Steps>
  <Step title="Define your check">
    Create a markdown file in `.continue/checks/` with a name, description, and prompt.

```markdown .continue/checks/security-review.md
---
name: Security Review
description: Flag hardcoded secrets and missing input validation
---

Review this pull request for security issues.

Flag as failing if any of these are true:

- Hardcoded API keys, tokens, or passwords in source files
- New API endpoints without input validation
- SQL queries built with string concatenation
- Sensitive data logged to stdout

If none of these issues are found, pass the check.
```
  </Step>
  <Step title="See the result">
    When a PR is opened, Continue runs each check and reports the result on GitHub.

    **Pass example:**
    ```
    ✅ Security Review — No issues found
    ```

    **Fail example:**
    ```
    ❌ Security Review — Issues found
    Fix: Remove hardcoded API key on line 23 of src/auth.js
    [Accept fix] [View details]
    ```
  </Step>
  <Step title="Accept or reject the fix">
    If a check fails, you can accept the AI-suggested fix directly from GitHub, or dismiss it. Continue learns from your feedback.
  </Step>
</Steps>

## Key Concepts

<CardGroup cols={2}>
  <Card title="Checks are markdown files" icon="file-text">

## Common Questions

<AccordionGroup>
  <Accordion title="How do checks run?">
    Checks run in CI via a GitHub Action that installs the `continue/check` CLI, reads your `.continue/checks/` files, and reports results as GitHub status checks.
  </Accordion>
  <Accordion title="What happens when a check fails?">
    The PR shows a red ❌ status on GitHub. Continue posts a comment with the failing check's name and an AI-generated fix suggestion. You can accept the fix directly or request changes.
  </Accordion>
  <Accordion title="Is my code/data private?">
    By default, code is sent to the AI model you configure. You can self-host models or use providers that don't store data. See [Running Continue Without Internet](/guides/running-continue-without-internet) for offline setups.
  </Accordion>
  <Accordion title="What models does Continue use?">
    Continue works with any model that supports function calling. By default it uses Anthropic's Claude, but you can configure any provider (OpenAI, Ollama, Azure, etc.). See [Model Providers](/customize/model-providers/overview).
  </Accordion>
</AccordionGroup>
